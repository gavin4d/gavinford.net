Translational drift

Getting the Robot Moving

First step is to spin up the robot. For now, this can easily be achieved by setting the motors to drive in opposite directions at a constant. Soon, we will look at a more advanced method using traction control.

So you have a spinning robot, let’s get it moving. As the robot spins around we need to pulse the motors faster as they go around one side and slower as they go around the other. The following image shows both wheel vectors and the net drift vector:

Implementation

Implementing translational drift requires the robot to know its heading and rotational velocity at all times. These values can be read directly or calculated from sensor inputs. In the next page we will look more in depth in an algorithm to combine sensor inputs called a Kalman filter.

The power applied to the wheels can take many different waveforms, but it is often done using a sinusoidal wave with the same frequency as the spin of the robot. Each motor’s power must be phase shifted by its angle on the robot.

Both wheel powers, \(P_L\) and \(P_R\), need a base power to maintain the robot’s spin. We can call this power \(P\): \[P_R= P\] \[P_L= P\] Now we can add the sinusoidal power to each one and phase shift it by the angle of the wheels: \[P_R=P+D_r cos(\theta)\] \[P_L=P+D_r cos(\theta + \pi)\] Where \(\theta\) is the heading of the robot, and \(D_r\) is the drift rate coefficient, which can be found experimentally. Generally a value of around 0.25 works well.
